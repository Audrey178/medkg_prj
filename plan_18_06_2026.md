## 4. Công việc đã hoàn thành (DONE ✅)

### 4.1 `scripts/run_medqa_eval.py`

Eval script cho MEDQA (USMLE-style 4-option MCQ).

**Tính năng:**
- Load MEDQA JSONL (`{"question", "options": {A/B/C/D}, "answer"}`)
- KG retrieval: exact + fuzzy n-gram matching (rapidfuzz) qua entity index
- Conditions: `base` (no KG) vs `kg` (with KG context)
- Metrics: Accuracy + 95% bootstrap CI
- Cảnh báo scope: ChronoMedKG cover rare diseases → entity_hit_rate thấp trên MEDQA tổng quát
- Lưu kết quả JSON vào `data/benchmark/medqa_results/`

**Usage:**
```bash
# Baseline (không cần KG data)
python -m scripts.run_medqa_eval --data data/medqa/test.jsonl --no-kg --n 100

# Full (local KG)
python -m scripts.run_medqa_eval --data data/medqa/test.jsonl --n 200

# Zenodo KG
python -m scripts.run_medqa_eval --data data/medqa/test.jsonl --kg-source zenodo

# Dry run (không tốn API)
python -m scripts.run_medqa_eval --data data/medqa/test.jsonl --dry-run
```

### 4.2 `scripts/run_pubmedqa_eval.py`

Eval script cho PubMedQA (yes/no/maybe).

**Tính năng:**
- Load `ori_pqal.json` (dict format) hoặc flat list variant
- MeSH terms từ metadata dùng làm entity seeds (precision cao hơn MEDQA)
- Conditions: `base` / `abstract_only` / `kg_only` / `kg+abstract`
- Metrics: Accuracy + Macro-F1 (3 classes) + 95% bootstrap CI
- Flag `--no-abstract`: test pure KG vs parametric knowledge (không có abstract)
- Lưu kết quả JSON vào `data/benchmark/pubmedqa_results/`

**Usage:**
```bash
# Baseline
python -m scripts.run_pubmedqa_eval --data data/pubmedqa/ori_pqal.json --no-kg --n 200

# Với abstract (standard PubMedQA setting)
python -m scripts.run_pubmedqa_eval --data data/pubmedqa/ori_pqal.json --n 500

# Pure KG vs parametric (không có abstract context)
python -m scripts.run_pubmedqa_eval --data data/pubmedqa/ori_pqal.json --no-abstract

# Dry run
python -m scripts.run_pubmedqa_eval --data data/pubmedqa/ori_pqal.json --dry-run
```
