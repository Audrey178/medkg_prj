# Knowledge Graph Retrieval-Augmented Generation Pipeline

Our KG-RAG implementation adapts the framework proposed by Soman et al. [31] with modifications tailored to leverage normalized knowledge graph structures. The pipeline consists of four sequential stages: entity extraction, entity matching, context retrieval, and answer generation.

## B.1 Large Language Model Configuration

We employ GPT-4o-mini accessed through OpenAI API for both entity extraction and answer generation tasks. The model is configured with a temperature of `0.1` to ensure reproducible outputs, with a maximum token limit of `4,096` tokens per response.

## B.2 Stage 1: Biomedical Entity Extraction

Given a user query \(Q\), we extract relevant biomedical entities using a prompted LLM. The extraction prompt instructs the model to identify entities across multiple biomedical categories including:

- Diseases
- Drugs
- Genes
- Proteins
- Biological processes
- Anatomical structures
- Medical procedures
- Symptoms
- Biomarkers

## B.3 Stage 2: Multi-Strategy Entity Matching

For each extracted entity \(e*i \in E*{extracted}\), we employ a cascaded matching strategy to identify corresponding nodes in our UMLS-normalized knowledge graph:

1. **Exact Matching**
   - String matching against entity names, UMLS preferred names, and CUI identifiers via Cypher queries.

2. **Abbreviation Expansion**
   - Automatic detection and expansion of abbreviation patterns using regular expression matching.

3. **Fuzzy String Matching**
   - Levenshtein distance ratio matching (implemented via the `FuzzyWuzzy` library) with a similarity threshold of **вүҘ 0.8**.

4. **Semantic Vector Search**
   - Approximate nearest neighbor search using entity embeddings generated by `sentence-transformers/all-MiniLM-L6-v2`, with a cosine similarity threshold of **вүҘ 0.7**.

Each matching strategy assigns confidence scores:

| Matching Strategy     | Confidence Score          |
| --------------------- | ------------------------- |
| Exact matching        | 1.0                       |
| Abbreviation matching | 0.95                      |
| Fuzzy matching        | Computed similarity ratio |
| Semantic matching     | Cosine similarity score   |

Candidates are ranked by confidence and deduplicated by CUI.

## B.4 Stage 3: Context Retrieval from Knowledge Graph

For each matched entity, we retrieve its local neighborhood from the Neo4j knowledge graph, including:

- Entity metadata:
  - Preferred name
  - CUI
  - Semantic types
  - Description
- Outgoing and incoming relationships
- Relationship attributes:
  - Conditions
  - Provenance documents
  - Temporal Information

The maximum number of relationships retrieved per entity is capped at **80**.

Retrieved facts are serialized into natural language statements.

When aggregated context exceeds the volume threshold (**150 sentences**), semantic similarity filtering is applied by computing cosine similarity between the query embedding and each context sentence embedding:

\[
\text{relevance}(s*i) = \cos(\mathbf{v}\_Q, \mathbf{v}*{s_i})
\tag{B1}
\]

where:

- \(\mathbf{v}\_Q\) is the embedding vector of the query.
- \(\mathbf{v}\_{s_i}\) is the embedding vector of sentence \(s_i\).

Only the top-\(k\) most relevant sentences are retained.

## B.5 Stage 4: Answer Generation

The final stage constructs prompts combining retrieved context with question-type-specific instructions. We define four prompt templates corresponding to BioASQ question types.

### Table B2. Prompt Templates for Different Question Types

| Question Type | Output Format                        | Key Instructions                  |
| ------------- | ------------------------------------ | --------------------------------- |
| Yes/No        | JSON (`answer`, `explanation`)       | Binary answer with evidence       |
| Factoid       | JSON (`answer`, `explanation`)       | Precise, concise answer           |
| List          | JSON (`answer` array, `explanation`) | Comprehensive, non-redundant list |
| Summary       | JSON (`answer`, `key points`)        | Coherent summary                  |

## B.6 Configuration Parameters

Key hyperparameters used in our KG-RAG pipeline are summarized below.

### Table B3. KG-RAG Pipeline Configuration Parameters

| Parameter             | Value              | Description                      |
| --------------------- | ------------------ | -------------------------------- |
| Context volume        | 150                | Maximum context sentences        |
| Similarity threshold  | 0.7                | Vector search distance threshold |
| Fuzzy match threshold | 0.8                | Minimum fuzzy match ratio        |
| Embedding model       | `all-MiniLM-L6-v2` | Sentence embedding model         |
| LLM temperature       | 0.1                | Generation temperature           |
